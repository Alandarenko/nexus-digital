Как ускорить сайт до идеального PageSpeed в 2026

NEXUS Digital · 13 февраля 2026

Скорость сайта напрямую влияет на конверсию, позиции в поиске и поведение пользователей. По данным Google, увеличение времени загрузки с 1 до 3 секунд повышает вероятность отказа на 32%. А при 5 секундах — на 90%.

В 2026 году и Google, и Яндекс используют метрики скорости как фактор ранжирования. Разбираем, как добиться идеального результата PageSpeed Insights.

Core Web Vitals в 2026 году

Google оценивает пользовательский опыт по трём ключевым метрикам:

Метрика Что измеряет Хорошо Плохо
LCP Загрузка основного контента ≤ 2.5 сек > 4.0 сек
INP Отзывчивость на действия ≤ 200 мс > 500 мс
CLS Визуальная стабильность ≤ 0.1 > 0.25

INP (Interaction to Next Paint) заменил FID в 2024 году и стал главной проблемой для многих сайтов. В отличие от FID, который измерял только задержку первого взаимодействия, INP учитывает все взаимодействия за время сессии.

Техники оптимизации

1. Critical CSS

Выделите CSS, необходимый для рендеринга первого экрана, и встройте его в <style> тега <head>. Остальные стили загружайте асинхронно. Это устраняет render-blocking и ускоряет LCP на 0.5–1.5 секунды.

2. Оптимизация изображений

  • Используйте WebP/AVIF — экономия 30–50% размера по сравнению с JPEG
  • Задавайте width и height для всех <img> — предотвращает CLS
  • Используйте srcset для адаптивных изображений
  • Сжимайте через Squoosh, ImageOptim или Sharp
  • Для hero-изображения используйте <link rel="preload">

3. Lazy Loading

Все изображения и iframe ниже первого экрана — с атрибутом loading="lazy". Для hero-изображения используйте loading="eager" и fetchpriority="high".

4. CDN (Content Delivery Network)

CDN размещает копии статики на серверах по всему миру. Для российской аудитории — Yandex Cloud CDN или Selectel CDN. Для глобальной — Cloudflare (бесплатный план). Улучшает TTFB на 50–80%.

5. Кеширование

Настройте заголовки Cache-Control для статических ресурсов:

  • Шрифты, изображения: max-age=31536000, immutable
  • CSS, JS (с хешами): max-age=31536000
  • HTML: no-cache (всегда проверять актуальность)

Service Worker — дополнительный уровень кеширования для PWA-сайтов. Стратегия network-first для HTML и cache-first для статики.

6. Оптимизация JavaScript

  • Defer и async — загружайте JS без блокировки рендеринга
  • Code splitting — разбивайте бандл на чанки, загружайте по требованию
  • Tree shaking — удаляйте неиспользуемый код при сборке
  • Минификация — Terser, esbuild или SWC
  • Избегайте тяжёлых библиотек — часто можно заменить на нативные API

Оптимизация INP

INP измеряет три компонента каждого взаимодействия:

  1. Input Delay — задержка до начала обработки. Минимизируйте работу в main thread, разбивайте длинные задачи через scheduler.yield()
  2. Processing Time — время выполнения обработчика. Оптимизируйте event listeners, избегайте forced reflow
  3. Presentation Delay — время до отрисовки. Минимизируйте DOM-мутации, используйте CSS-анимации вместо JS

Главное правило: любая задача в main thread не должна превышать 50 мс. Используйте requestAnimationFrame для визуальных обновлений и requestIdleCallback для фоновых задач.

Яндекс и скорость

Яндекс не использует Core Web Vitals напрямую, но учитывает поведенческие факторы, которые напрямую зависят от скорости:

  • Показатель отказов — медленный сайт = больше отказов = снижение позиций
  • Время на сайте — если страница не загрузилась, пользователь уходит
  • Глубина просмотра — тормозящий сайт не мотивирует открывать другие страницы
  • Возвращаемость — быстрый сайт запоминается и получает повторные визиты

Яндекс.Метрика показывает скорость загрузки в отчёте «Время загрузки страниц» — используйте его для мониторинга.

Yandex Cloud CDN

Для сайтов с российской аудиторией Yandex Cloud CDN — оптимальное решение. Серверы расположены в Москве, Санкт-Петербурге, Владивостоке и других городах. Интеграция с Object Storage для статики. Поддержка HTTP/2 и Brotli-сжатия.

Чеклист оптимизации PageSpeed

  • Critical CSS встроен в <head>
  • Шрифты загружаются через font-display: swap и preconnect
  • Изображения в WebP/AVIF с указанием размеров
  • Lazy loading для контента ниже fold
  • JavaScript с defer/async, минифицирован
  • Gzip или Brotli сжатие на сервере
  • HTTP/2 или HTTP/3 включён
  • CDN для статических ресурсов
  • Service Worker для PWA-кеширования
  • Cache-Control заголовки для всех ресурсов
  • Нет render-blocking CSS/JS
  • DOM-дерево менее 1500 элементов
  • Нет layout shifts (CLS ≤ 0.1)
  • Все обработчики событий завершаются за < 50 мс
  • Мониторинг через Яндекс.Метрику и CrUX
